<DetailSection
  label="Overview"
  title="Text-to-SQL only became useful after the data model was made explicit."
  class="detail-section-results"
>
  <DetailColumns>
    <DetailItem title="Context">
      <DetailCopy>
        <p>Ad-hoc analytics requests were consuming too much analyst time. In practice, the slow part was not SQL syntax. It was finding the right tables, understanding what columns meant, mapping business logic correctly, and figuring out join patterns that could actually be trusted.</p>
        <p>That made even routine questions expensive. Complex requests often took 4 to 10 hours, which limited how many requests the team could realistically support each month.</p>
      </DetailCopy>
    </DetailItem>
    <DetailItem title="Approach">
      <DetailCopy>
        <p>I built a structured metadata layer in YAML that captured how analysts actually reason about the data model: table definitions, column semantics, keys, partitioning, join patterns, and business terminology.</p>
        <p>The workflow then used AI coding agents to generate SQL, execute it in Athena, and validate results before anything was handed back. The leverage came from structuring context well enough that the system could reuse it repeatedly.</p>
      </DetailCopy>
    </DetailItem>
  </DetailColumns>
</DetailSection>

<DetailSection
  label="Workflow"
  title="A practical validation loop around AI-generated SQL"
  class="detail-section-takeaways"
  intro="The workflow narrowed context first, then generated, executed, and checked SQL before any result was handed back to a human."
>
  <DetailStack>
    <DetailItem title="Scoped context">
      <p>Tables were grouped by domain and narrowed through index files so the agent only saw the relevant slice of metadata.</p>
    </DetailItem>
    <DetailItem title="SQL generation">
      <p>AI coding agents used the structured metadata to generate and revise SQL based on the actual business question.</p>
    </DetailItem>
    <DetailItem title="Execution and QA">
      <p>The workflow ran queries in Athena, checked execution success, and validated results before handing anything back to a human.</p>
    </DetailItem>
    <DetailItem title="Reusable knowledge">
      <p>The resulting structure could be updated and reused so later requests started from a better base instead of from scratch.</p>
    </DetailItem>
  </DetailStack>
</DetailSection>
