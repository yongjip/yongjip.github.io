<section class="detail-section detail-section-results">
  <p class="section-label">Overview</p>
  <h2>The project started from a simple question: how should recommendation logic stay grounded?</h2>
  <div class="detail-columns">
    <article class="detail-item">
      <h3>Context</h3>
      <p>
        The idea was to build a web application that suggested dietary supplements based
        on profile inputs such as age, gender, lifestyle, habits, goals, and pregnancy
        status.
      </p>
      <p>
        I wanted the recommendation flow to be informed by actual product and review data
        rather than by a fixed list of generic suggestions.
      </p>
    </article>
    <article class="detail-item">
      <h3>Approach</h3>
      <p>
        I collected public product information and review data, then used that material to
        shape a recommendation prototype that could connect user profile inputs to a more
        concrete product set.
      </p>
      <p>
        The project was exploratory rather than clinical. The point was to build a usable
        recommendation flow, not to present medical advice.
      </p>
    </article>
  </div>
</section>

<section class="detail-section detail-section-takeaways">
  <p class="section-label">Workflow</p>
  <h2>The prototype linked questionnaire inputs to a searchable product base.</h2>
  <div class="detail-stack">
    <article class="detail-item">
      <h3>Profile inputs</h3>
      <p>
        The interface captured user traits and goals that could influence which supplement
        categories looked relevant.
      </p>
    </article>
    <article class="detail-item">
      <h3>Public product data</h3>
      <p>
        Public product information and review text were collected so the recommendation
        flow could be tied to a real catalog instead of abstract examples.
      </p>
    </article>
    <article class="detail-item">
      <h3>Recommendation logic</h3>
      <p>
        The prototype connected the profile and catalog layers so the user could move from
        questionnaire answers to a narrower set of options.
      </p>
    </article>
  </div>
</section>

<section class="detail-section">
  <p class="section-label">Takeaways</p>
  <h2>Recommendation projects improve once the data and the user flow are designed together.</h2>
  <div class="detail-stack">
    <article class="detail-item">
      <h3>What mattered</h3>
      <p>
        The interesting part was linking messy real-world catalog information to a simple
        questionnaire-driven interface.
      </p>
    </article>
    <article class="detail-item">
      <h3>Why it was useful</h3>
      <p>
