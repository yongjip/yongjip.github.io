케이스 스터디

AI 기반 Text-to-SQL 시스템으로 데이터 접근성을 확장한 방법

반복되던 데이터 탐색 과정을 AI가 활용할 수 있는 재사용 가능한 지식 시스템으로 바꿔, 애드혹 분석 시간을 4~10시간에서 1시간 미만으로 줄였습니다.

핵심은 SQL 생성 자체보다도, 분석가의 이해 과정을 재사용 가능한 구조로 만들고 실행과 검증까지 하나의 워크플로우로 묶는 것이었습니다.

개요

병목은 SQL 문법이 아니라 데이터 이해였습니다.

문제 배경

애드혹 요청은 계속 늘어났고, 분석가의 시간이 과도하게 소모되고 있었습니다. 반복되는 일의 핵심은 쿼리 자체보다도 어떤 테이블을 써야 하는지, 컬럼 의미가 무엇인지, 비즈니스 로직을 데이터 모델에 어떻게 맞춰야 하는지, 어떤 조인 패턴이 신뢰 가능한지 다시 파악하는 과정이었습니다.

그래서 단순한 질문도 비용이 컸고, 복잡한 요청은 4~10시간이 걸리기도 했습니다. 결과적으로 한 달에 처리할 수 있는 요청 수가 구조적으로 제한됐습니다.

접근 방식

분석가가 실제로 데이터를 이해하는 방식을 YAML 기반 지식 레이어로 구조화했습니다. 여기에는 테이블 정의, 컬럼 의미, 키와 파티셔닝 정보, 조인 패턴, 비즈니스 용어가 포함됩니다.

핵심은 모델 자체가 아니라, AI가 반복해서 사용할 수 있도록 맥락을 구조화하는 것이었습니다.

워크플로우

AI가 만든 SQL을 실제 검증 루프로 감싼 구조

필요한 메타데이터 범위를 먼저 좁히고, SQL 생성 뒤에는 실행과 검증을 붙여 신뢰 가능한 결과만 전달하도록 만들었습니다.

맥락 범위 축소

테이블을 도메인별로 그룹화하고 인덱스 파일로 먼저 좁혀서, 에이전트가 필요한 메타데이터만 보도록 만들었습니다.

SQL 생성

AI 코딩 에이전트가 구조화된 메타데이터를 바탕으로 실제 비즈니스 질문에 맞는 SQL을 생성하고 수정했습니다.

실행과 QA

워크플로우는 Athena에서 쿼리를 실행하고, 실행 성공 여부와 결과 품질을 확인한 뒤에만 사람에게 전달했습니다.

지식의 재사용

한 번 구조화한 지식은 다음 요청에서 다시 쓸 수 있어, 매번 처음부터 데이터 모델을 복원하는 비용을 줄일 수 있었습니다.

시스템 흐름

질문에서 검증된 결과까지

flowchart TD A[사용자 질문] --> B[도메인과 테이블 선택] B --> C[지식 레이어] C --> C1[테이블 정의] C --> C2[컬럼 의미] C --> C3[조인 패턴] C --> C4[비즈니스 용어] C --> D[AI 에이전트] D --> E[SQL 쿼리] E --> F[Athena 실행] F --> G{실행 성공} G -- 아니오 --> D G -- 예 --> H[결과 검증] H --> I{품질 통과} I -- 아니오 --> D I -- 예 --> J[사람에게 전달] J --> K[지식 업데이트] K --> C

이 시스템은 SQL 생성에서 끝나지 않았습니다. 실행, 검증, 반복 수정을 거쳐 신뢰 가능한 결과만 전달했습니다.

결과

구조화된 맥락이 속도와 처리량을 함께 바꿨습니다.

도입 전

4~10시간

일반적인 애드혹 분석 소요 시간.

도입 후

1시간 미만

구조화된 맥락과 검증 루프 도입 후 소요 시간.

처리량

20건에서 50건+

월간 애드혹 분석 처리 역량의 대략적 변화.

배운 점

실제 시스템의 핵심은 지식 아키텍처였습니다.

어려운 건 이해 과정

분석 업무의 비용 대부분은 계산보다도, 비즈니스가 데이터 모델에 어떻게 대응되는지 매번 다시 복원하는 데서 발생합니다.

단순한 구조가 확장된다

이 상황에서는 무거운 플랫폼보다도, 깔끔한 YAML 지식 레이어와 실행 및 QA 루프가 더 큰 레버리지를 만들었습니다.

AI는 규율과 함께 쓸 때 강해진다

출력은 제약과 검증이 있을 때 비로소 실무에 유용해집니다. 그리고 그 결과가 다시 팀의 재사용 가능한 지식으로 돌아갈 때 가치가 커집니다.
