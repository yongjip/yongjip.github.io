케이스 스터디

Graph 기반 창고 최적화 및 시뮬레이션 워크플로우 구축

창고 graph 를 만드는 도구와 단순화된 시뮬레이션 모델을 구축했고, 현재는 이를 더 현실적인 운영 분석으로 확장하는 작업을 이어가고 있습니다.

출발점

storage node 데이터는 있었지만, 활용 가능한 graph 는 없었습니다.

좌표를 포함한 storage node 정보는 있었지만, 작업자가 실제로 이동할 수 있는 경로를 반영한 실용적인 graph 는 없었습니다. 그래서 레이아웃, 접근성, 재고 배치를 구조적으로 평가하기가 어려웠습니다.

즉시 필요한 것은 최적화 자체보다도, 경로 기반 분석을 가능하게 만드는 모델을 먼저 구축하는 일이었습니다.

현재 범위

완성형 최적화 엔진이 아니라 graph 구성 도구와 단순화된 시뮬레이션입니다.

이 프로젝트를 완성된 창고 최적화 시스템으로 소개하고 싶지는 않습니다. 현재 시스템은 두 부분으로 구성됩니다. 하나는 warehouse graph 를 만드는 도구이고, 다른 하나는 재고 재배치 효과를 추정하는 단순화된 시뮬레이션 모델입니다.

목표는 완벽한 디지털 트윈이 아니라 방향성 있는 의사결정 지원입니다.

Graph 구성

storage node 와 이동 경로를 분석 가능한 warehouse graph 로 바꾸는 과정

node grouping

먼저 storage node 를 그룹화합니다. 현재는 Manhattan distance 기반으로 실용적인 group 을 만들고, 이후 경로 계산을 더 큰 단위에서 다룰 수 있도록 하고 있습니다.

access point 설정

각 group 에 하나 이상의 access point 를 두어, 작업자가 해당 영역에 어떻게 접근하는지 graph 상에서 표현할 수 있게 했습니다.

path 기반 연결

사용자가 실제 이동 가능한 path 를 그리면, 시스템이 path 와 가까운 access point 를 자동으로 연결해 graph 구조를 생성합니다.

Grouping 의 의미

창고를 최적화하기 전에 경로 계산 비용부터 줄여야 했습니다.

모든 storage node 단위에서 바로 최단경로를 계산하면 비용이 매우 커집니다. grouping 은 graph 복잡도를 줄여 경로 계산 문제를 다루기 쉽게 만듭니다. 그 덕분에 이후의 시뮬레이션과 최적화 작업도 계산량과 워크플로우 측면에서 훨씬 현실적인 형태가 됩니다.

실제로 grouping 레이어는 단순 전처리가 아니라, graph 크기와 path computation 비용을 제어하는 시스템 설계의 핵심입니다.

시뮬레이션

재고 재배치 아이디어를 빠르게 시험하기 위한 단순화된 모델

규칙 기반 등급화

상품과 storage node 를 완전한 운영 모델 대신 단순화된 규칙 기반 로직으로 등급화했습니다.

재배치 실험

그 등급에 따라 재고를 다시 배치하고, 재배치 전후 작업자 이동거리의 추정값을 비교합니다.

방향성 중심 결과

이 결과는 정확한 최종 운영 수치를 주장하기 위한 것이 아니라, 더 나은 배치가 이동거리를 얼마나 줄일 수 있는지 설명하기 위한 것입니다.

약 50%

단순화된 시뮬레이션에서 추정된 이동거리 감소 폭

확정 아님

이 수치는 방향성 결과이며 실제 창고 제약을 완전히 반영하지는 않습니다

주요 리스크

피킹이 특정 로케이션에 몰리면 국소적인 병목이 발생할 수 있습니다

해석

중요한 것은 숫자 자체보다 설명 가능한 구조입니다.

유의미한 신호

단순화된 시뮬레이션만으로도 inventory placement 가 이동거리에 큰 영향을 줄 수 있다는 가능성을 보여줍니다.

중요한 한계

현재 모델은 혼잡이나 실제 작업 제약을 충분히 반영하지 못하므로, 50%라는 수치를 최종 운영 효과로 해석하지는 않습니다.

핵심 의미

그럼에도 graph 기반 모델을 통해 더 나은 재고 배치가 왜 이동거리를 줄이는지 구조적으로 설명할 수 있게 됐다는 점이 중요합니다.
